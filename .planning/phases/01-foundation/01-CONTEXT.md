# Phase 1: Foundation (Фундамент) - Context

**Gathered:** 2026-02-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Инициализированный проект Next.js 15 с настроенными шрифтами, TypeScript-типами, константами свадебных данных и базовыми UI-компонентами. Это фундамент — скелет проекта, на котором строятся все последующие фазы. Секции контента, динамические компоненты и интеграции — отдельные фазы.

</domain>

<decisions>
## Implementation Decisions

### Шрифты и типографика
- **Единый шрифт: Calmius Extra Light** — для заголовков, имён и основного текста
- Базовый размер шрифта на мобильном: 18px
- Letter-spacing заголовков: стандартное (как задумано автором шрифта)
- Иконки Lucide: тонкие (strokeWidth: 1)

### Визуальный стиль — сдержанная элегантность
- Романтичная классика — но без перегруза декором
- Элегантность через типографику и пространство, минимум декоративных элементов
- Лёгкая бумажная/льняная текстура на фоне — как настоящее приглашение (глобально в layout)
- Общий фон: тёплый кремовый (#FFF8F0 — #FFFDF5)
- Основной цвет текста: тёмно-коричневый/сепия (не чёрный) — гармония с кремовым
- Плавный поток секций — без резких переходов, разделение через отступы
- Средние отступы между секциями (48-64px)
- Тон текста: тёплый и личный, на «вы» строчное

### Компоненты — стилистика
- Скругление углов: умеренное (8-12px)
- Тени карточек: едва заметные — глубина без тяжести
- Кнопки основные: контурные (outline), при hover — плавная заливка цветом
- Кнопка CTA (RSVP «Подтвердить»): заливка (filled), большой размер
- Два размера кнопок: обычный и большой
- Два стиля кнопок: контурная (основная) + заливка (для акцентных действий)
- Ссылки: в акцентном цвете без подчёркивания, подчёркивание при hover

### Цветовая палитра
- **Основной акцент: Александрит (#598c74)** — для кнопок, линий, точек-индикаторов, активных элементов, сообщения успеха
- **Основной текст: Тёмный шоколад (#3C1518)** — вместо чёрного, для всего текста и иконок Lucide
- **Фон: Тёплый кремовый (#FFF8F0 — #FFFDF5)** — как бумажное приглашение
- **Светлый александрит** (с opacity) — для акцентных фонов, неактивных точек-индикаторов, декоративных линий
- **Бежевый/песочный** — нейтральный для чередования фонов секций
- Контурные кнопки: рамка и текст в александрите, при hover заливка александритом
- Иконки Lucide: в цвете шоколада (#3C1518), сдержанные
- Ошибки валидации: тёплый красный/терракотовый — заметно, но в палитре
- Placeholder в полях: стандартный серый

### Hero-секция
- Полноэкранная (100vh)
- Тёмный overlay ~30-40% — фото хорошо видно, текст читается за счёт text-shadow
- Текст по центру экрана
- Порядок элементов: Имена → Дата → Фраза-приглашение
- Имена: «Анна & Михаил» (амперсанд, только имена)
- Дата прописью: «15 августа 2026»
- Фраза: поэтичная (например, «Приглашаем вас разделить с нами этот особенный день»)
- Parallax-эффект: заметный, выраженная глубина
- Плавное проявление (fade-in ~1-1.5s) при первой загрузке
- Два варианта фото: desktop + mobile (в constants.ts) — object-fit cover
- Загрузка: белый экран → плавное появление (без skeleton)

### Анимации
- Появление секций: fade-up (снизу вверх) — основной вариант, но несколько вариантов через prop (fade-up, fade-in, fade-left/right)
- Скорость: медленные и плавные (~0.8-1s) — романтичный ритм
- Срабатывают каждый раз при скролле в область видимости (не once: true)
- Порог срабатывания: рано (~10-20% видимости)

### Навигация
- Точки-индикаторы справа (компонент `SectionDots`)
- Кликабельные — плавный скролл к секции (~1200ms)
- Активная точка: заполнена александритом
- Неактивные: светлый александрит (с opacity)
- Скрыты на мобильных

### Структура данных (constants.ts)
- Отдельные экспорты по секциям: `heroData`, `timelineData`, `dressCodeData` и т.д.
- Имена пары: только имена (не фамилии)
- Реальные данные — все конкретные значения будут предоставлены
- Язык: только русский
- Таймзона: одна, захардкоженная (Europe/Moscow)

### Порядок секций на странице
1. Hero
2. Countdown
3. Инфо «О торжестве» (Где/Когда)
4. Программа дня «Как пройдёт наш день»
5. Дресс-код
6. RSVP «Ждём вашего ответа»
7. Карта «Место проведения»
8. Footer

### Заголовки секций
- Стиль: компонент `SectionHeading` — текст + декоративные линии по бокам (с утончением к краям)
- Цвет линий: светлый александрит
- Countdown: без заголовка
- Инфо: «О торжестве»
- Программа: «Как пройдёт наш день»
- Дресс-код: «Дресс-код»
- RSVP: «Ждём вашего ответа»
- Карта: «Место проведения»

### Секция Countdown
- Крупные числа в ряд, подпись под каждой (дни | часы | минуты | секунды)
- Без разделителя между блоками (пробел)
- Полные слова с правильным склонением: «1 день», «2 дня», «5 дней»
- После свадьбы: заменить текстом «Спасибо, что были с нами!»
- Без заголовка

### Секция «О торжестве» (Где/Когда)
- Общий заголовок «О торжестве» над двумя карточками
- Карточки: иконка сверху, текст по центру
- В ряд на десктопе, стопкой на мобильном
- «Когда»: дата + время сбора гостей
- «Где»: название места + адрес
- Одно место проведения (церемония + банкет)

### Секция «Программа дня»
- Таймлайн: тонкая непрерывная вертикальная линия с тематическими иконками Lucide
- ~5 этапов: сбор гостей, церемония, банкет, торт, завершение
- Каждый этап: время + название (без описания)
- Зигзаг (чередование) на десктопе, одна сторона на мобильном
- Маппинг иконок к этапам — в компоненте, не в данных

### Секция «Дресс-код»
- Короткая фраза дресс-кода
- Ряд из 4-5 цветных кружков палитры с названиями цветов
- Только текст + палитра, без пожеланий

### Секция RSVP
- Поля: Имя (обязательное), Фамилия (обязательное), Статус участия, Чекбокс «Приду с парой» (только при «Приду»), Пожелания (обязательное)
- Статус: два варианта «Приду» / «Не смогу прийти» — кнопки-переключатели (toggle buttons)
- Чекбокс «Приду с парой»: тоже кнопка-переключатель (единый стиль)
- Поля ввода: полная рамка (классические)
- Placeholder в поле пожеланий: «Напишите пожелание молодожёнам»
- Валидация: в реальном времени (onBlur), красная рамка + текст под полем
- Кнопка «Подтвердить»: filled, большая
- Загрузка: спиннер внутри кнопки
- Успех: форма заменяется сообщением, персонализировано по ответу:
  - «Приду» → «Спасибо! Мы рады, что вы будете с нами» (в александрите)
  - «Не смогу» → «Спасибо за ответ, будем скучать!»
- Ошибка: «Что-то пошло не так. Свяжитесь с нами» + телефон + Telegram/WhatsApp ссылка
- Повторный визит: localStorage, показывать «Вы уже ответили»
- Все компоненты формы внутри RSVP, не выносить в отдельные базовые компоненты

### Секция «Карта»
- Iframe Яндекс.Карт, квадратный (~400x400px)
- Кнопка «Построить маршрут» → Яндекс.Карты (контурная)
- Адрес текстом под картой (дублирует карточку «Где»)

### Footer
- Минимальный: «Анна & Михаил · 2026»

### Заголовок и мета
- Title вкладки: имена пары
- Favicon: эмодзи-сердце
- OG-описание: «Анна & Михаил приглашают вас — 15 августа 2026»

### Адаптивность
- Три брейкпоинта: mobile (до 640px), tablet (640-1024px), desktop (1024+)
- Ширина контента на десктопе: средняя (max-width ~768px), центрирован
- Только светлая тема

### Контакт для ошибки RSVP
- Телефон + Telegram/WhatsApp ссылка — хранится в constants.ts, используется только при ошибке RSVP

### Claude's Discretion
- Конкретные hex-коды для бежевого/песочного, светлого александрита, тёплого красного ошибок
- Стиль кнопки filled (CTA) — александрит или шоколад
- Точный оттенок кремового фона
- Точный оттенок тёмно-коричневого текста (в направлении #3C1518)
- Конкретный текст фразы-приглашения на Hero (поэтичный, тёплый тон)
- Размеры заголовков (крупнее) vs основной текст — дифференциация через size/weight при едином шрифте Calmius
- Текст замены countdown после свадьбы
- Текстура фона — конкретная реализация бумажной/льняной текстуры
- Spacing и padding внутри компонентов
- Exact parallax intensity
- Loading skeleton — не нужен (белый экран → fade-in)
- Error state handling для компонентов

</decisions>

<specifics>
## Specific Ideas

- Общая эстетика: «как настоящее бумажное приглашение» — кремовый фон, бумажная текстура, сдержанный декор
- **Шрифт Calmius Extra Light** — единый для всей страницы, дифференциация заголовков через размер/weight
- Цвета свадьбы: александрит (#598c74) + тёмный шоколад darkbloom (#3C1518) — необычная, благородная палитра
- Hero: фото занимает весь экран, текст умеренного размера — фото тоже должно «дышать»
- Таймлайн: зигзаг с тематическими иконками — визуально интересно на десктопе
- Линии при заголовках: с утончением к краям — изящный, а не грубый разделитель
- Кнопки: контурные с плавной заливкой при hover — утончённый переход
- Тон текстов: тёплый и личный, на «вы» строчное — не формальный, но уважительный

</specifics>

<decisions>
## Implementation Decisions

### Countdown — визуальный стиль
- Карточки с числами: дни, часы, минуты, секунды — каждая единица в отдельном блоке
- Стиль карточек — в стиле сайта, используют существующий Card компонент с alexandrite-акцентами и тенями
- С секундами — все 4 единицы отображаются
- Подписи на русском с правильным склонением (1 день, 2 дня, 5 дней; 1 час, 2 часа, 5 часов и т.д.)

### Countdown — заголовок
- Заголовок «До нашего дня» над карточками

### Countdown — поведение
- При достижении нуля — заменить карточки текстом «Сегодня наш день!» или аналогичной праздничной фразой
- Плавная fade-анимация при смене каждой цифры (каждую секунду)

### Countdown — появление
- Stagger-эффект при первом появлении: карточки появляются по очереди (дни → часы → минуты → секунды) с небольшой задержкой

### Анимации при скролле — общий стиль
- Единый fade-up для всех секций (как сейчас), без разнообразия вариантов по секциям
- Текущие настройки AnimatedSection сохраняются: duration 0.9s, viewport.once: false, amount: 0.15

### Parallax-эффекты
- Parallax на Hero и countdown — фон движется медленнее при скролле
- Выраженный эффект глубины — фон на ~50% медленнее контента

### Доступность
- `prefers-reduced-motion` — полностью отключить все анимации: никаких fade, parallax, stagger. Контент сразу видимый.

### Галерея — решение
- Фотогалерея убрана из проекта полностью (не отложена, а удалена)
- Требование CONT-06 снимается

### Claude's Discretion
- Точная реализация parallax (CSS transform vs Framer Motion useScroll)
- Праздничная фраза при достижении нуля countdown
- Точные значения stagger delay между карточками
- Точные easing-функции для fade-анимации цифр

</decisions>

<specifics>
## Specific Ideas

- Карточки countdown должны визуально вписываться в существующую дизайн-систему (Card, alexandrite-цвет, тени)
- Склонение русских слов для единиц времени — обязательно корректное (не «1 дней»)
- Parallax должен ощущаться как глубина, а не как «сломанный скролл»

# Phase 2: Static Sections (Статические секции) - Context

**Gathered:** 2026-02-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Все информационные секции страницы отображают контент из constants.ts. Превращаем placeholder-разметку в полноценные визуальные компоненты: Hero, инфо-карточки, таймлайн программы, дресс-код, footer. Без динамики (countdown — Phase 3), без интеграций (RSVP, карта — Phase 4).

</domain>

<decisions>
## Implementation Decisions

### Hero-секция
- Текст по центру экрана, поверх фонового фото
- Полупрозрачная плашка glassmorphism (backdrop-blur + белый полупрозрачный) за текстом для читаемости
- Без реального фото — фон из SVG feTurbulence текстуры бумаги (уже есть в проекте)
- Порядок элементов сверху вниз: **имена → фраза → дата**
- Одно фото адаптивно через `object-fit: cover` (поля Desktop/Mobile в типах остаются, но на практике одно изображение)
- Без скролл-индикатора (стрелка вниз не нужна)

### Таймлайн программы дня
- **Зигзаг (лево-право)** — события чередуются слева и справа от вертикальной линии
- На мобильном зигзаг сворачивается в прямую линию (всё слева от линии)
- **Уникальная Lucide-иконка на каждый этап** — бокал, кольца, торт и т.д.
- **Декоративный стиль** — линия с утончением, иконки в цветных кружках, тени на карточках
- Линия: **сплошная, цвет александрит** — единый стиль с SectionHeading
- Кружки с иконками: **средние (28-32px)**, белый фон, обводка александрит, иконка цветная
- Каждый этап обёрнут в **Card-компонент** (белая плашка с тенью)
- **Добавить описания к этапам** — расширить тип `TimelineEvent` полем `description`, дополнить constants.ts (1 строка на этап)

### Инфо-карточки «Когда» / «Где»
- **Крупная иконка сверху** — Lucide (Calendar / MapPin) над заголовком, центрированная
- Расположение: **одна колонка, друг под другом** — каждая карточка на всю ширину, больше воздуха

### Дресс-код
- **Только текст + палитра** — без дополнительных иконок и иллюстраций
- Цветовые кружки: **крупнее (56-64px)** — заметный акцент секции, подписи под ними

### Footer
- **Только копирайт** — «Анна & Михаил · 2026», без контактов, без «Сделано с ♥»
- Тонкая линия сверху — градиентная в стиле SectionHeading (александрит)
- Мелкий текст, пониженная непрозрачность — не привлекает внимание
- Без иконок-декора
- Просто последний блок страницы (не sticky)
- Фон **чуть отличается** от основного — лёгкий контраст, визуальное завершение
- Компактные отступы (py-4/py-6)

### Claude's Discretion
- Размер имён в Hero — подобрать баланс под плашку и фото
- Конкретный оттенок фона footer — на усмотрение (перекликается с палитрой свадьбы)
- Конкретные Lucide-иконки для каждого этапа таймлайна
- Placeholder-описания для этапов таймлайна (до получения реальных данных)

</decisions>

<specifics>
## Specific Ideas

- Hero glassmorphism: backdrop-blur + полупрозрачный белый блок, фото видно по краям — современный свадебный стиль
- Таймлайн зигзаг: декоративный, с Card-блоками — каждый этап выглядит как мини-карточка рядом с линией
- Инфо-карточки в одну колонку: больше воздуха, каждая «дышит», крупная иконка сверху
- Дресс-код: кружки 56-64px — палитра как визуальный акцент, а не мелкая деталь

</specifics>

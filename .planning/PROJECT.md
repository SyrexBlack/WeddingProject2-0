# Wedding Invitation (Свадебная визитка)

## What This Is

Персональный свадебный веб-сайт (одностраничная визитка-приглашение) для пары. Сайт содержит всю информацию о торжестве — дату, время, место, программу дня, дресс-код, галерею фотографий, интерактивную карту и форму RSVP для подтверждения участия гостей. Ориентирован на мобильные устройства (гости получают ссылку через мессенджеры).

## Core Value

Гость открывает ссылку из мессенджера, видит красивое приглашение со всей информацией и может подтвердить своё участие в один клик.

## Requirements

### Validated

- ✓ Hero-секция с именами пары, датой и фоновым фото — Phase 2
- ✓ Информационные блоки: «Где», «Когда», «Дресс-код», «Программа дня» — Phase 2

### Active

- [ ] Обратный отсчёт (таймер) до начала торжества
- [ ] Интерактивная карта (Yandex Maps или Google Maps) с меткой места проведения
- [ ] Галерея фотографий пары
- [ ] Форма RSVP: имя, подтверждение участия, выбор блюд (опционально)
- [ ] Обработка RSVP без бэкенда (EmailJS, Formspree или Google Таблица)
- [ ] Open Graph теги для красивого превью в Telegram/WhatsApp
- [ ] Оптимизация изображений для быстрой загрузки на мобильных
- [ ] Анимации при скролле (Framer Motion)
- [ ] Деплой на Vercel с HTTPS

### Out of Scope

- Мультиязычность — визитка на одном языке (русский)
- Личный кабинет / авторизация гостей — не нужна для одностраничника
- Мобильное приложение — только веб
- CMS / админ-панель — контент статический
- Платёжная система — сбор денег не через сайт

## Context

**Тип проекта:** Greenfield — одностраничный свадебный сайт-визитка.

**Референсные шаблоны:**
1. [nextjs-wedding-invite](https://github.com/wzulfikar/nextjs-wedding-invite) — Next.js, RSVP, красивые секции
2. [wedding-invitation](https://github.com/LeeKyuHyuk/wedding-invitation) — React, корейский минимализм
3. [sakeenah](https://github.com/mrofisr/sakeenah) — React, современная типографика
4. [react-wedding-card](https://github.com/uyu423/react-wedding-card) — React, простой и надёжный

**Стиль:** Минимализм / современный. Шрифты и анимации определяются на основе референсов.

**Целевая аудитория:** Гости свадьбы, получающие ссылку через Telegram/WhatsApp. Преимущественно мобильные устройства.

**Репозиторий GitHub:** https://github.com/SyrexBlack/WeddingProject2-0

## Constraints

- **Стек:** Next.js (база), Tailwind CSS (стилизация), Framer Motion (анимации), Lucide React (иконки)
- **Бэкенд:** Без собственного сервера — только бессерверные решения (EmailJS/Formspree/Google Sheets)
- **Хостинг:** Vercel или Netlify (бесплатный тарифный план, HTTPS из коробки)
- **Бюджет:** $0 — бесплатные сервисы и хостинг
- **Срок:** До свадьбы (конкретная дата уточняется)
- **Устройства:** Mobile-first, но корректное отображение на десктопе

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Next.js как основа | SSG для скорости, экосистема React, деплой на Vercel | ✓ Working (Next.js 16) |
| Tailwind CSS для стилизации | Быстрая разработка, mobile-first утилиты | ✓ Working (Tailwind 4) |
| Framer Motion для анимаций | Декларативный API, хорошая интеграция с React | ✓ Working (AnimatedSection) |
| Без бэкенда (serverless формы) | Нулевой бюджет, простота поддержки | — Pending |
| Референс-шаблоны для вдохновения | Изучить лучшие практики из 4 шаблонов, не копировать | ✓ Applied |
| Glassmorphism для Hero overlay | backdrop-blur-md + bg-white/30, элегантный стиль поверх текстуры | ✓ Shipped Phase 2 |
| SVG feTurbulence для текстуры фона | Inline SVG шум вместо PNG — ноль HTTP-запросов | ✓ Shipped Phase 1 |
| Zigzag timeline layout | Чередование лево-право на десктопе, линейный на мобильном | ✓ Shipped Phase 2 |

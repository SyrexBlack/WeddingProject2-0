# Feature Research

**Domain:** Свадебный пригласительный одностраничный сайт (wedding invitation one-page website)
**Researched:** 2026-02-22
**Confidence:** HIGH

---

## Feature Landscape

### Table Stakes (Гости ожидают это)

Это базовые функции, без которых пригласительный сайт теряет смысл. Гости уйдут или будут разочарованы, если этого нет.

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| **Hero-секция** (имена пары, дата, фото) | Первое впечатление; гость должен за 2 секунды понять: кто женится и когда | Низкая | Полноэкранный блок с фоновым изображением, имена пары, дата свадьбы. Все 4 референса имеют hero-секцию |
| **Обратный отсчёт (countdown)** | Создаёт ощущение предвкушения; стандарт жанра | Низкая | Дни/часы/минуты/секунды до события. Автоматическое переключение на "Свадьба состоялась!" после даты |
| **Информация о месте проведения** | Гости обязаны знать, куда ехать | Низкая | Название заведения, адрес, этаж/зал, контактный телефон площадки |
| **Дата и время** | Критически важная информация | Низкая | Чёткое отображение даты, дня недели, времени начала церемонии и банкета |
| **Интерактивная карта** | 70%+ гостей откроют сайт с телефона и нажмут "Построить маршрут" | Средняя | Яндекс.Карты (для РФ-аудитории) или Google Maps. Кнопка "Открыть в навигаторе". Sakeenah использует Google Maps |
| **RSVP-форма** | Пара должна знать количество гостей для рассадки и кейтеринга | Средняя | Имя, подтверждение присутствия (приду/не приду/+1), пожелания. Serverless бэкенд (EmailJS / Formspree / Google Sheets) |
| **Мобильная адаптивность (responsive)** | 80-90% гостей откроют ссылку в мессенджере на телефоне | Средняя | Mobile-first подход. Все элементы корректно отображаются от 320px. Все 4 референса мобильно-адаптивны |
| **OG-теги для соцсетей** | При отправке ссылки в WhatsApp/Telegram/VK должна показаться красивая превью-карточка | Низкая | og:title, og:description, og:image, og:url. Все 4 референса поддерживают OG-теги |
| **Фотогалерея** | Визуальная привлекательность; гости хотят видеть фото пары | Средняя | 6-12 фотографий. Лайтбокс при клике. Оптимизация изображений (lazy loading, WebP) |
| **Дресс-код** | Гости часто спрашивают "что надеть" — информация на сайте снимает вопросы | Низкая | Текстовый блок с иконками. Палитра цветов, если есть цветовая тема свадьбы |
| **Программа/расписание** | Гости должны понимать тайминг дня: церемония, банкет, развлечения | Низкая | Вертикальный таймлайн с иконками и временем каждого этапа |

### Differentiators (Запоминающиеся детали)

Эти функции превращают сайт из "ещё одного шаблона" в нечто, что гости запомнят и покажут друзьям.

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| **Scroll-анимации (Framer Motion / AOS)** | Элегантное появление секций при прокрутке создаёт премиальное ощущение | Средняя | Fade-in, slide-up для секций. Sakeenah использует Framer Motion — это лучший выбор для React |
| **Параллакс-эффекты** | Глубина и объём страницы; выделяет среди плоских шаблонов | Средняя | Фоновые изображения двигаются с другой скоростью. Не переусердствовать — на мобильных может тормозить |
| **Таймлайн "Наша история любви"** | Персонализация; гости узнают историю пары через ключевые даты | Средняя | Горизонтальный или вертикальный таймлайн: "Познакомились", "Первое свидание", "Предложение" |
| **Фоновая музыка** | Эмоциональное усиление; создаёт атмосферу | Низкая | Кнопка play/pause обязательна. Автоплей запрещён браузерами — нужен "Нажмите для звука". Sakeenah и wedding-invitation (Korean) имеют музыку |
| **Персонализированные имена через URL-параметры** | Каждый гость видит "Дорогой Иван!" вместо безликого приглашения | Средняя | `?to=base64(name)` в URL. Имя подставляется в hero и RSVP-форму. nextjs-wedding-invite и sakeenah поддерживают это |
| **Конфетти/частицы при открытии** | Wow-эффект в первые секунды; запоминается | Низкая | canvas-confetti или tsparticles. Запускать один раз при первом открытии, не зацикливать |
| **Плавные переходы между секциями** | Ощущение нативного приложения, а не веб-страницы | Средняя | Smooth scroll, секционная навигация. CSS scroll-snap опционально |
| **"Открыть приглашение" (конверт-анимация)** | Имитирует реальное открытие бумажного приглашения; запускает музыку | Средняя | Полноэкранная заставка с кнопкой "Открыть". При клике — анимация раскрытия конверта, старт музыки. Sakeenah использует этот паттерн |
| **Кнопка "Добавить в календарь"** | Практичная польза; гость не забудет дату | Низкая | Генерация .ics файла или ссылки Google Calendar. Одна кнопка решает проблему |
| **Пожелания/гостевая книга** | Интерактивность; гости оставляют тёплые слова до свадьбы | Средняя | Форма + отображение предыдущих пожеланий. Sakeenah хранит в PostgreSQL; для MVP хватит Google Sheets |
| **Шеринг через мессенджеры** | Удобство пересылки приглашения между гостями | Низкая | Кнопки: WhatsApp, Telegram, копировать ссылку. wedding-invitation (Korean) имеет KakaoTalk шеринг |

### Anti-Features (НЕ делать)

Осознанное решение НЕ строить следующие функции. Каждая из них увеличивает сложность без пропорциональной пользы для персонального свадебного сайта.

| Feature | Why Requested | Why Problematic | Alternative |
|---------|---------------|-----------------|-------------|
| **Аутентификация/логин гостей** | "Защитить сайт от посторонних" | Трение при открытии; гости не будут регистрироваться ради приглашения. Потеря 50%+ аудитории | Персонализация через URL-параметры (без логина) |
| **CMS/админ-панель** | "Редактировать контент без кода" | Это one-time сайт для одной свадьбы. Стоимость разработки CMS многократно превышает пользу | Конфигурационный JSON/TS файл с данными свадьбы |
| **Реал-тайм чат** | "Общение гостей между собой" | Модерация, спам, серверные затраты. Есть WhatsApp/Telegram-группы | Ссылка на общий чат в мессенджере |
| **Платёжная система** | "Сбор денежных подарков онлайн" | Юридические сложности, комиссии, безопасность финансовых данных | Текстовый блок с реквизитами/QR-кодом для перевода (как в wedding-invitation Korean) |
| **Мультиязычность (i18n)** | "Гости из разных стран" | Значительное усложнение для персонального сайта. nextjs-wedding-invite имеет i18n, но для РФ-свадьбы избыточно | Один язык (русский). При необходимости — второй через ручное дублирование |
| **Система email-уведомлений** | "Автоматические напоминания гостям" | Требует email-сервис, шаблоны, отписки, GDPR. Избыточно | Ручная рассылка через мессенджеры. RSVP-данные в Google Sheets |
| **Дашборд управления гостями** | "Отслеживать кто подтвердил" | Полноценный бэкенд, база данных, интерфейс управления. Sakeenah делает это, но она — платформа, не персональный сайт | Google Sheets как "дашборд" — RSVP-ответы автоматически попадают в таблицу |
| **Мульти-страничность (роутинг)** | "Отдельные страницы для каждой секции" | Увеличивает время загрузки, разрывает повествовательный поток одностраничника | Smooth scroll между секциями с якорными ссылками |
| **Комплексная фотогалерея с загрузкой** | "Гости загружают свои фото" | Хранилище, модерация, NSFW-фильтр, затраты на CDN | Ссылка на Google Photos альбом с общим доступом |

## Feature Dependencies

```
Hero-секция
  +-- Данные пары (имена, дата, фото)
  +-- OG-теги (используют те же данные)
  +-- Персонализация через URL (?to=name)

Обратный отсчёт
  +-- Дата свадьбы (из конфига)

Информация о месте
  +-- Интерактивная карта (встраивается в секцию)
  +-- Кнопка "Добавить в календарь" (использует дату + адрес)

RSVP-форма
  +-- Serverless бэкенд (EmailJS / Formspree / Google Sheets)
  +-- Персонализация через URL (пре-заполнение имени)

Фотогалерея
  +-- Оптимизация изображений (lazy loading, WebP)
  +-- Лайтбокс-компонент

Scroll-анимации
  +-- Все секции (анимации применяются к каждой секции)
  +-- Intersection Observer API или Framer Motion

"Открыть приглашение" (конверт)
  +-- Фоновая музыка (запуск при открытии)
  +-- Hero-секция (показ после анимации)

Пожелания/гостевая книга
  +-- Serverless бэкенд (тот же, что RSVP, или отдельный endpoint)
```

### Граф зависимостей (упрощённый)

```
[Конфиг данных свадьбы]
       |
       +---> [Hero-секция] ---> [OG-теги]
       |          |
       |          +---> [Персонализация URL]
       |
       +---> [Обратный отсчёт]
       |
       +---> [Место + Карта] ---> [Добавить в календарь]
       |
       +---> [Расписание / Дресс-код]
       |
       +---> [Фотогалерея] ---> [Лайтбокс]
       |
       +---> [RSVP-форма] ---> [Serverless бэкенд]
       |
       +---> [Scroll-анимации] (обёртка над всеми секциями)
       |
       +---> [Конверт-анимация] ---> [Музыка]
```

## MVP Definition

### Launch With (v1) — Запуск

Минимальный набор для рабочего пригласительного. Цель: гость получает ссылку, узнаёт всё необходимое, подтверждает присутствие.

1. **Hero-секция** — имена, дата, главное фото
2. **Обратный отсчёт** — дни/часы/минуты/секунды
3. **Информация о месте** — адрес, зал, время
4. **Интерактивная карта** — Яндекс.Карты или Google Maps
5. **Расписание/программа** — таймлайн дня
6. **Дресс-код** — текст + палитра цветов
7. **Фотогалерея** — 6-10 фото с лайтбоксом
8. **RSVP-форма** — имя, присутствие, +1, пожелания → Google Sheets
9. **OG-теги** — красивая превью-карточка при шеринге
10. **Mobile-first респонсив** — корректная работа на всех устройствах
11. **Базовые scroll-анимации** — fade-in при появлении секций

### Add After Validation (v1.x) — После запуска

Добавить после того, как v1 проверен на реальных гостях и работает стабильно.

1. **Персонализация через URL** — `?to=base64(Иван Иванов)` → "Дорогой Иван!"
2. **"Открыть приглашение"** — конверт-анимация на входе
3. **Фоновая музыка** — с кнопкой play/pause
4. **Конфетти при открытии** — canvas-confetti
5. **Кнопка "Добавить в календарь"** — .ics файл + Google Calendar ссылка
6. **Шеринг через мессенджеры** — WhatsApp, Telegram, копировать ссылку
7. **Параллакс-эффекты** — на hero и разделителях между секциями

### Future Consideration (v2+) — На будущее

Если будет желание и время, но не критично для свадьбы.

1. **Таймлайн "Наша история"** — ключевые даты отношений
2. **Гостевая книга** — пожелания с отображением на сайте
3. **Переключение тем** — светлая/тёмная
4. **Расширенные анимации** — Lottie-анимации, SVG-морфинг
5. **PWA-режим** — оффлайн-доступ (полезно на площадке без интернета)

### Ключевые выводы из анализа конкурентов:

1. **nextjs-wedding-invite** — самый конфигурируемый (JSON-конфиг, CSV гостей, i18n), но минимален по визуальным эффектам. Хорошая база для fork.
2. **wedding-invitation (Korean)** — самый популярный (157 звёзд). Фокус на корейских реалиях (KakaoTalk, банковские счета). Минималистичный, чистый дизайн.
3. **sakeenah** — самый функционально богатый. Полноценная платформа с PostgreSQL, multi-tenancy, Framer Motion анимациями. Избыточен для персонального сайта, но лучший источник вдохновения для UX-паттернов.
4. **react-wedding-card** — самый простой и надёжный. Хорош как отправная точка; поддерживает комментарии (LiveRe/Facebook) и Google Maps API.

### Что взять от каждого:
- От **nextjs-wedding-invite**: паттерн конфигурации, персонализация через URL
- От **wedding-invitation**: минимализм дизайна, шеринг в мессенджерах, блок реквизитов
- От **sakeenah**: Framer Motion анимации, конверт-анимация, обратный отсчёт, архитектура компонентов
- От **react-wedding-card**: простота, OG-теги, Google Maps интеграция, галерея
